Stint “Calm” · Warm Paper

One shared language for every surface — the sidebar, Timer, Entries, Clients, Reports, Settings, the entry editor, the time-range picker, the menu-bar tray popover, and the focused flows (software update, merge, sleep review). Built on Fluent 2 fundamentals (light, depth, material restraint, motion, scale) with the calm density of Linear and the native grammar of Apple HIG. The goal, per issue #29: less visual noise, more fluency, total consistency — a quiet canvas and a single rationed accent. These mockups are the source of truth for style: the app conforms to them, and style is verified by comparing rendered output to them, never by asserting CSS in code (see process.html).

Principles

One rationed accent on a quiet canvas. The surfaces are a near-neutral monochrome so the eye lands on the content; a single accent marks the one thing that matters most in a view — its primary or create action, and the live running state. It is a signal, never decoration. (The palette is the tokens' to set — colour is a system decision, not a per-view one.)

Depth, not tint. A region rises by a soft shadow alone, never a coloured or bordered box. Surfaces don't nest, and no sub-region earns a fill of its own — elevation is the only thing that says "this is a layer".

The UI speaks for itself. Every element earns its pixels. When shape, position, or state already carry the meaning, the words go — no headings, captions, instructions, or state words the layout makes plain — and a verb that recurs becomes its icon. If something still seems to need explaining, redesign it or move the detail behind an affordance instead of printing prose on the surface.

Small type ramp, tabular time. Four roles do the work; hierarchy comes from size + weight (450 / 590 / 680), never colour or italics. Every clock and duration uses tabular numerals so digits never jitter.

4px grid, generous air. Every pad, gap, and height is a multiple of 4, with roomy margins. Radii are a fixed trio — 8px controls, 12px cards, 16px window & overlays.

One control per choice-shape. Toggle = on/off · segmented = ≤4 exclusive peers · select = many options. Running state is a small green dot. Line icons only — never emoji.

Fluent foundations

Microsoft's Fluent Design System rests on five environmental elements — light, depth, motion, material, scale — under three Fluent 2 principles — familiarity, inclusivity, consistency. Stint keeps the discipline of each but dials the showy parts (acrylic blur, reveal glow, parallax) down to suit a calm, local, single-user desktop tool. Here is what each means and how this system applies it.

Light — draw the eye, don't decorate. Fluent uses light to guide attention (reveal highlight, focus glow). Our only "light" is the clay focus ring (--ring) and the warm hover wash (--hover). The single primary action is the brightest thing in a view; everything else stays quiet so the eye always lands in one place.

Depth — a strict elevation ladder. Fluent splits an app into a base canvas with content floating above on z-ordered shadows. We keep one ladder: canvas → card (--sh-card) → raised → popover (--sh-pop) → modal (--sh-modal). The tray popover, merge dialog and update banner each sit exactly one rung above what they cover. Never nest cards.

Motion — functional, short, rare. Fluent uses connected animation and parallax for continuity. We keep motion to ~120ms state changes: toggles slide, a chosen segment lifts, the running dot is the one persistent cue. No parallax, no decorative transitions, nothing that moves without meaning.

Material — warm paper, not glass. Fluent's acrylic and mica make surfaces translucent. We swap translucency for opaque warm paper on an ivory canvas; the lone nod to material is the menu bar's faint blur where it meets the OS. Material restraint means no tinted, dashed, or decorative boxes — surface is signalled by elevation alone.

Scale — one language, many sizes, on a 4px grid. Fluent scales a single design across form factors from a four-pixel base unit. The same tokens, type ramp, controls and icons run unchanged from the 332px menu-bar tray popover to the 1040px main window — every pad, gap and height a multiple of 4, every corner one of the fixed radius trio (8 controls / 12 cards / 16 window & overlays).

Familiarity (Fluent 2 principle) — build on what the user already knows. Native macOS window chrome and the system font stack mean Stint feels like the platform it runs on rather than a web page in a frame.

Inclusivity (Fluent 2 principle) — hierarchy never rides on colour alone. Size and weight carry structure, so it survives colour-blindness and dimmed screens; every semantic state pairs its colour with a word or icon (the run dot beside "running", the worded warn pill). Tabular numerals keep durations legible and steady.

Consistency (Fluent 2 principle) — one of everything. One token set, one line-icon family, one control per choice-shape, one elevation ladder, applied identically across every surface in this set. That single-source discipline is the entire point of this document: a new view should be assemblable from these parts without inventing anything.
